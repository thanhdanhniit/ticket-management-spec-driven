# 1. Generate UI Specs
## Prompt — VisionAgent (Screenshot → UI Structure JSON)
**Model**: Gemini FLASH

the VisionAgent skill to analyze the provided UI screenshot.

Objective:
Extract the UI layout and components from the screenshot and generate a structured UI representation.

Instructions:
- Identify the page name if visible.
- Identify page type (list page, form page, dashboard, detail page).
- Identify layout sections such as header, sidebar, toolbar, filters, content area, tables, forms, and pagination.
- Detect UI components including buttons, inputs, dropdowns, tables, cards, modals, and tabs.
- Extract visible field names and table column names.
- Identify visible user actions such as create, edit, delete, search, filter, and navigation.

Output:
Return ONLY the UI Structure JSON generated by the VisionAgent skill and save it into specs/services/.

Input Screenshot:
{{SCREENSHOT}}

## Prompt — UISpecArchitect skill
**Model**: Gemini 3.1 Pro High

Use the UISpecArchitect skill to generate a UI specification. Recommend to reference to shared global sidebar and shared top navigation bar ui-specs.

Objective:
Convert the provided UI Structure JSON into a complete UI specification document.

Inputs:
UI Structure JSON:
{{UI_STRUCTURE_JSON}}

Optional reference screenshot:
{{SCREENSHOT}}

Instructions:
- Expand the UI structure into a detailed specification.
- Describe page purpose and layout.
- Describe UI components and their behavior.
- Define visible fields and data.
- Describe user actions.
- Describe tables, forms, filters, and pagination.
- Describe navigation if visible.
- Describe error states and empty states.

Focus only on UI behavior and interactions.
Do NOT describe backend logic.

Output:
Return the UI specification document generated by the UISpecArchitect skill save it into specs/services/ folfder.
